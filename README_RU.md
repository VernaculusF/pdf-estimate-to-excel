# PDF сметы в Excel

Инструмент берет PDF-файлы со сметами из папки `input` и сохраняет Excel-файлы в папку `output`.

## Как пользоваться

1. Положить PDF-файлы в папку `input`.
2. Запустить `start.bat` двойным кликом.
3. Забрать результаты из папки `output`.

После обработки в `output` будут:

- отдельный `.xlsx` для каждого PDF;
- `conversion_report.xlsx` - понятный отчет по сохранности символов;
- `conversion_report.json` - тот же отчет в техническом формате.

В каждом Excel-файле первый лист `Смета` содержит распознанную таблицу. Дополнительный лист `Исходный текст` хранит полный текст PDF или OCR-оценку для сканов, чтобы символы не терялись даже там, где таблица распознана неидеально.

## Что отвечать заказчику про 5000 символов

Если текст уже извлечен из PDF и записывается в Excel, 5000 символов в одной ячейке не режутся. Excel поддерживает до 32767 символов в ячейке, а в проекте есть проверка, что строка на 5000 символов сохраняется и читается обратно без потерь.

Риск потерь возникает раньше: на этапе чтения PDF. Поэтому после конвертации создается отчет `conversion_report.xlsx`. В нем видно:

- сколько видимых символов найдено в исходном PDF;
- сколько символов оказалось в Excel;
- сколько символов совпало;
- сколько символов не найдено в Excel;
- процент сохранности.

Отдельные колонки `table_*` показывают качество именно табличного листа. Основные колонки `preservation_percent` и `lost_visible_chars` показывают сохранность по всей книге Excel с учетом листа `Исходный текст`.

Для обычных PDF с текстовым слоем отчет считается по тексту PDF. Для сканов без текстового слоя отчет помечает источник как `ocr_estimate`, потому что там точность зависит от качества скана и OCR.

## Запуск из командной строки

Обработать всю папку:

```bat
venv\Scripts\python.exe main.py --input input --output output
```

Обработать один файл:

```bat
venv\Scripts\python.exe main.py --file input\example.pdf --output output
```

Принудительно использовать OCR:

```bat
venv\Scripts\python.exe main.py --input input --output output --use-ocr
```
